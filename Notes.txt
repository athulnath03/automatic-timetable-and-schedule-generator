# 🚫 Why Vercel / Netlify won’t run your current setup

Vercel & Netlify:

* ❌ **Do NOT run long-lived servers** (`python app.py`)
* ❌ No listening on ports (5000, 8000, etc.)
* ❌ No background processes
* ✅ Only support **serverless functions**

Your Flask app is a **stateful server**, so it cannot run directly.

---

## 🚀 STEP-BY-STEP (RECOMMENDED)

---

## 🔹 STEP 1 — Prepare Frontend for Production

### In `frontend/`:

```bash
npm run build
```

This creates:

```
frontend/dist/
```

---

## 🔹 STEP 2 — Host FRONTEND on Vercel

### 📁 Frontend changes

Create `frontend/vercel.json`:

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/" }
  ]
}
```

### 📤 Deploy

```bash
cd frontend
vercel
```

✔ Output:

```
https://automatic-timetable.vercel.app
```

---

## 🔹 STEP 3 — Host BACKEND on Render

### 📁 Backend changes

Create `backend/requirements.txt` (if not already):

```
flask
flask-cors
gunicorn
```

Create `backend/start.sh`:

```bash
gunicorn app:app
```

---

### 📤 Deploy on Render

1. Go to [https://render.com](https://render.com)
2. New → Web Service
3. Connect GitHub repo
4. Root directory: `backend`
5. Build command:

   ```
   pip install -r requirements.txt
   ```
6. Start command:

   ```
   gunicorn app:app
   ```

✔ Backend URL:

```
https://timetable-api.onrender.com
```

---

## 🔹 STEP 4 — Connect Frontend → Backend

In React (`GeneratePage.jsx`):

```js
axios.post(
  "https://timetable-api.onrender.com/api/generate",
  payload
)
```

Or better:

### 🔐 Use environment variable

```bash
VITE_API_URL=https://timetable-api.onrender.com
```

```js
axios.post(`${import.meta.env.VITE_API_URL}/api/generate`, payload)
```
